## 2. ユーザーフロー

```
初回アクセス
    → APIキー入力欄が空 → ユーザーがAPIキーを入力
    → カテゴリを選択（デフォルト: 化粧品）
    → 画像をドラッグ&ドロップ または ファイル選択
    → プレビューに画像が表示される
    → 「チェック実行」ボタンをクリック
    → ローディング表示（「AIが確認中です...」）
    → 結果が表示される
        ├ 全体判定バナー
        └ 項目別テーブル（記載あり✓/記載なし✗/判定不可?）
    → 履歴に自動保存
    → 必要に応じてCSVエクスポート
```

## 3. UIコンポーネント仕様

### [A] 設定エリア

#### APIキー入力
- `input[type=password]` で表示はマスク
- placeholder: `sk-ant-...`
- ページ読み込み時に localStorage から自動読み込み
- 変更のたびに localStorage に保存

#### カテゴリ選択
- ラジオボタン形式（2択）
- 選択肢: `化粧品（薬機法第61条）` / `医薬部外品（薬機法第59条）`
- デフォルト: 化粧品
- 切り替え時に以前の結果表示はリセット

---

### [B] 画像アップロードエリア

#### ドラッグ&ドロップゾーン
- 点線枠でビジュアルを表現
- ドラッグオーバー時: 枠の色を変える（ハイライト）
- テキスト: 「画像をドラッグ&ドロップ、またはクリックして選択」
- クリックでファイル選択ダイアログを開く

#### ファイル選択（隠しinput）
- `input[type=file][multiple][accept="image/*"]`
- 複数ファイル選択可

#### プレビュー一覧
- サムネイル形式（横100px程度）で選択済み画像を表示
- 各サムネイルに削除ボタン（×）
- ファイル名を下部に表示

---

### [C] チェック実行ボタン

- テキスト: 「チェック実行」（通常時）
- テキスト: 「確認中...」（API呼び出し中、disabled状態）
- 無効条件: APIキー未入力 または 画像未選択

---

### [D] 結果表示エリア

#### 全体判定バナー
- 全項目 `found` → 緑色バナー「問題なし：全ての必須項目が確認されました」
- 1件でも `not_found` → 赤色バナー「要確認：記載が確認できない項目があります」
- `unclear` のみ → 黄色バナー「要確認：判定できなかった項目があります」

#### 項目別結果テーブル

| # | 項目名 | 根拠条文 | 判定結果 | 備考 |
|---|---|---|---|---|
| 1 | 製品名（名称） | 第61条第1号 | ✓ 記載あり | |
| 2 | ... | | ✗ 記載なし | |

- `found` → ✓（緑）
- `not_found` → ✗（赤）
- `unclear` → ?（黄）

---

### [E] 履歴エリア

#### CSVエクスポートボタン
- テキスト: 「履歴をCSVでエクスポート」
- 履歴が空の場合は非活性

#### 履歴一覧テーブル

| チェック日時 | カテゴリ | 画像枚数 | 全体判定 | 詳細 |
|---|---|---|---|---|
| 2026-05-19 21:30 | 化粧品 | 3枚 | 要確認 | [展開▼] |

- 最新50件を表示
- 各行をクリックすると詳細（項目別結果）を展開表示

---

### フッター（免責表示）

```
【免責事項】
本ツールによる判定はAIによる参考情報です。
最終的な薬機法への適合判断は、必ず薬事専門家または行政機関にご確認ください。
当ツールの判定結果を根拠とした法的責任は負いかねます。
```

常時表示（スクロールしても見える位置）。
