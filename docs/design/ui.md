## 2. ユーザーフロー

```
初回アクセス
    → APIキー入力欄が空 → ユーザーがAPIキーを入力
    → カテゴリを選択（デフォルト: 化粧品）
    → 画像をドラッグ&ドロップ または ファイル選択
    → プレビューに画像が表示される
    → 「チェック実行」ボタンをクリック
    → ローディング表示（「AIが確認中です...」）
    → 結果が表示される
        ├ 全体判定バナー
        └ 項目別テーブル（記載あり✓/記載なし✗/判定不可?）
    → 履歴に自動保存
    → 必要に応じてCSVエクスポート
```

## 3. UIコンポーネント仕様

### [A] 設定エリア

#### APIキー入力
- `input[type=password]` で表示はマスク
- placeholder: `sk-ant-...`
- ページ読み込み時に localStorage から自動読み込み
- 変更のたびに localStorage に保存

#### カテゴリ選択
- ラジオボタン形式（2択）
- 選択肢: `化粧品（薬機法第61条）` / `医薬部外品（薬機法第59条）`
- デフォルト: 化粧品
- 切り替え時に以前の結果表示はリセット

---

### [B] 画像アップロードエリア

#### ドラッグ&ドロップゾーン
- 点線枠でビジュアルを表現
- ドラッグオーバー時: 枠の色を変える（ハイライト）
- テキスト: 「画像をドラッグ&ドロップ、またはクリックして選択」
- クリックでファイル選択ダイアログを開く

#### ファイル選択（隠しinput）
- `input[type=file][multiple][accept="image/*"]`
- 複数ファイル選択可

#### プレビュー一覧
- サムネイル形式（横100px程度）で選択済み画像を表示
- 各サムネイルに削除ボタン（×）
- ファイル名を下部に表示

---

### [C] チェック実行ボタン

- テキスト: 「チェック実行」（通常時）
- テキスト: 「確認中...」（API呼び出し中、disabled状態）
- 無効条件: APIキー未入力 または 画像未選択

---

### [D] 結果表示エリア

#### 全体判定バナー
- 全項目 `found` → 緑色バナー「問題なし：全ての必須項目が確認されました」
- 1件でも `not_found` → 赤色バナー「要確認：記載が確認できない項目があります」
- `unclear` のみ → 黄色バナー「要確認：判定できなかった項目があります」

#### 項目別結果テーブル

| # | 項目名 | 根拠条文 | 判定結果 | 備考 |
|---|---|---|---|---|
| 1 | 製品名（名称） | 第61条第1号 | ✓ 記載あり | |
| 2 | ... | | ✗ 記載なし | |

- `found` → ✓（緑）
- `not_found` → ✗（赤）
- `unclear` → ?（黄）

---

### [E] 履歴エリア

#### CSVエクスポートボタン
- テキスト: 「履歴をCSVでエクスポート」
- 履歴が空の場合は非活性

#### 履歴一覧テーブル

| チェック日時 | カテゴリ | 画像枚数 | 全体判定 | 詳細 |
|---|---|---|---|---|
| 2026-05-19 21:30 | 化粧品 | 3枚 | 要確認 | [展開▼] |

- 最新50件を表示
- 各行をクリックすると詳細（項目別結果）を展開表示

## 6. デバイス別動作状況と改良案

### 6.1 現状評価

| デバイス | 動作状況 | 備考 |
|---|---|---|
| PC ブラウザ | 問題なし | 設計通り |
| タブレット | ほぼ問題なし | `max-width: 960px` でセンタリング、テーブルは横スクロール可、タップ操作も動作 |
| スマートフォン | 使えるが不便な点あり | 下記2点を参照 |

### 6.2 スマートフォンでの課題

| # | 課題 | 内容 |
|---|---|---|
| SP-01 | ドロップゾーンの文言 | 「ドラッグ＆ドロップ」とあるがスマホではタップのみ有効（動作はする） |
| SP-02 | 結果・履歴テーブル | 5〜6列あるため横スクロールが必要になる場合がある |

### 6.3 改良案

#### SP-01 対応: ドロップゾーン文言のデバイス切り替え

**実装方針**: `navigator.maxTouchPoints` でタッチデバイスを判定し、文言を動的に切り替える。

```js
// JS での判定例
const isTouch = navigator.maxTouchPoints > 0;
dropZoneText.textContent = isTouch
  ? 'タップして画像を選択'
  : '画像をドラッグ＆ドロップ、またはクリックして選択';
```

CSS `@media (hover: none)` との併用で、ドラッグ関連の視覚効果（ホバー強調）もスマホでは非表示にできる。

#### SP-02 対応: 結果テーブルのカード型レイアウト

**実装方針**: `@media (max-width: 640px)` でテーブルをカード型に切り替える。

```css
@media (max-width: 640px) {
  .result-table thead { display: none; }
  .result-table tr {
    display: block;
    border: 1px solid var(--border);
    border-radius: var(--radius);
    margin-bottom: 12px;
    padding: 12px;
  }
  .result-table td {
    display: flex;
    justify-content: space-between;
    border: none;
    padding: 4px 0;
  }
  .result-table td::before {
    content: attr(data-label);
    font-weight: 600;
    color: var(--subtext);
  }
}
```

HTML 側で `<td data-label="項目名">` のように `data-label` 属性を付与する必要がある。

### 6.4 優先度

| 改良案 | 優先度 | 工数 |
|---|---|---|
| SP-01: ドロップゾーン文言切り替え | 中 | 小（JS 数行 + CSS 数行） |
| SP-02: 結果テーブルのカード型化 | 中 | 中（HTML 属性追加 + CSS） |
