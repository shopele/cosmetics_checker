## Phase 1: 精度・説明性の強化（〜1.5週）

**目標**: 「NG表示だけで終わらない」体験を実現し、市場の有料システムに肩を並べる説明性を確保する。

### 1.1 判定理由・代替案の詳細化

**対応提案書項目**: 1.1

**実装内容**:
- `js/rules.js` の各チェック項目に以下フィールドを追加:
  - `reason`: なぜ必要か（法的根拠の説明）
  - `law_reference`: 該当条文
  - `suggestion`: 改善例文
- `api/check.js` のシステムプロンプトに「理由と改善案も JSON で返す」指示を追加
- `js/app.js` の結果テーブルに折りたたみ式の詳細表示を追加

**変更ファイル**:
- `js/rules.js`
- `api/check.js`（プロンプト変更）
- `js/app.js`（結果表示ロジック・`parseResponse()` 修正）
- `css/style.css`（折りたたみスタイル）

**完了条件**:
- [x] 各チェック項目の `reason` / `law_reference` / `suggestion` が定義されている
- [x] 判定不可・記載なし時に折りたたみで詳細・改善案が表示される
- [x] 既存の記載あり表示に影響がない

---

### 3.2 PDF 対応（分割送信前提）

**実装内容**:
- `pdfjs-dist` を動的インポート（PDF 選択時のみ読み込む）:
  ```js
  const pdfjsLib = await import('pdfjs-dist');
  ```
- 戦略:
  - テキストレイヤーあり → テキスト抽出のみで完結
  - テキストレイヤーなし → ページ単位で画像化、サイズ見積もり後に自動分割送信
- 既存の `resizeAndEncode()` を拡張してペイロードサイズを見積もる
- 上限超過時に「○ページずつに分けて送信します」と UI に明示

**変更ファイル**:
- `js/app.js`（PDF 読み込み・分割ロジック追加）
- `index.html`（`accept` 属性に `.pdf` 追加）
- `css/style.css`

**完了条件**:
- [x] PDF がアップロードできる
- [x] テキストレイヤーがある PDF でテキスト抽出が動作する
- [x] 大きな PDF で自動分割送信が動作する
- [x] 初期ロードに `pdfjs-dist` のサイズ影響がない
