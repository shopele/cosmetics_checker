## 1. AI検索・リサーチツール（全般的な情報収集）

インターネット上の膨大な情報を横断的に調査し、まとめやマインドマップ、レポートを作成するためのAIプラットフォームの比較です。

| ツール | 特徴 | 主な強み |
|---|---|---|
| **Perplexity** | リアルタイム検索とソース引用 | 正確性98.2%、ビジネス・学術的ファクトチェックに最適 |
| **Genspark** | 「Sparkpage」と呼ばれるまとめページを自動生成 | ビジュアル重視の情報俯瞰に適している |
| **Felo** | 多言語の情報を日本語に翻訳・集約 | マインドマップやプレゼンスライド自動構築に優れている |
| **Deep Research（ChatGPT）** | 1回の指示で数十回以上の自律検索を実行 | 論理的思考と深い分析が強み |
| **Deep Research（Gemini）** | 1回の指示で数十回以上の自律検索を実行 | 多角的な情報整理、Googleドキュメント連携に強い |

## 2. 化粧品・医薬部外品 「薬機法・表示チェック」特化システム

化粧品や医薬部外品のパッケージ（本体・外箱）における薬機法違反表現の検知や成分表示照合を行う高度なシステム群です。

### 2.1 パッケージ文字照合・全成分表示チェック特化型

| システム | 提供企業 | 主な機能 |
|---|---|---|
| **PROOFREADER** | クラフト・ビュー | デザイン版下の成分リストを薬事申請済み全成分データと自動比較。並び順「順不同」にも対応 |
| **版下チェックデザイナー** | ユニオン技術センター | Excelから登録した原稿とパッケージ版下データを照合。誤記・抜けをハイライト |
| **review-it! for Package** | 凸版印刷 | 表示原稿と制作中パッケージPDFのデザインを自動比較。修正前後の差分チェック可能 |
| **フォルトファインダープロ** | Too | パッケージ向け文字検版支援ソフト。変形レイアウト（扇型・円形）の照合、AIOCR対応 |

### 2.2 パッケージ表現・薬機法リスク判定特化型

| システム | 提供企業 | 主な機能 |
|---|---|---|
| **薬機法チェック LLMアプリ** | Allganize | 厚労省規制情報に基づいてAIがリスク自動判定。理由提示・代替案生成も対応 |
| **TooAI SUITEパッケージチェックシステム** | Too | 各企業に最適化した個別ルール（薬事法禁止表現・企業固有ルール）でチェック。クラウド構築 |

## 4. 分析・考察

### 4.1 画像チェック対応の重要性

- **有料システム（B2B）**: ほぼ全てが画像解析（OCR）機能を搭載。実務レベルの本体・パッケージ外箱チェックに対応
- **無料ツール**: テキスト入力限定がほとんど。手軽な情報収集や簡易チェックには利便性がある

### 4.2 用途別の選択基準

| 用途 | 推奨システム | 理由 |
|---|---|---|
| **実務的なパッケージ校正** | PROOFREADER / review-it! for Package | 成分データ照合・版下校正に特化、精度が高い |
| **デザイン変形レイアウト対応** | フォルトファインダープロ | 扇型・円形など複雑なレイアウトにも対応 |
| **薬機法リスク判定・代替案提案** | 薬機法チェック LLMアプリ / TRUSQUETTA AD | 生成AIが解説・改善案を提示 |
| **広告表現チェック（Web・SNS）** | ADJUDGE / 広告チェックAI | オンライン広告の包括的チェック |
| **低コスト簡易チェック** | セルフ薬機法チェック / 薬事法広告表現チェック | 無料で基本的な違反検出が可能 |

### 4.3 本プロジェクトとの比較ポイント

**本プロジェクト（化粧品・医薬部外品 表記チェッカー）の位置づけ**:

- ✓ **Claude API（Vision）を活用した画像解析**: PROOFREADER や薬機法チェック LLMアプリ と同様に、パッケージ画像から OCR で情報抽出が可能
- ✓ **判定結果の履歴管理**: 有料システムには搭載例が少ない機能
- ✓ **複数モデル選択**: Opus/Sonnet/Haiku の柔軟性は市場にない
- ✓ **カスタム項目対応**: 社内基準の追加・削除が容易
- ✗ 有料システム（PROOFREADER など）の「薬事申請済みデータベースとの照合」機能は未実装
