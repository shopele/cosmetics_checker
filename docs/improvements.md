## 優先度 高（実用性・安定性に直結）

### [x] 1. 画像圧縮の実装

現状はリサイズ（1920px）のみで、高解像度 JPEG は数 MB になる場合がある。Canvas で JPEG 品質を下げて送信データを削減し、API コスト削減・タイムアウト軽減につながる。

**実装内容**: `js/app.js` の `resizeAndEncode()` で元ファイルサイズに応じて JPEG 品質を 0.85 / 0.75 / 0.65 に動的調整。`MAX_IMAGE_PX` を 1920 → 1600 に下げてデータ量削減。

### [x] 2. API エンドポイントのレート制限

`api/check.js` に IP ベースのインメモリレート制限を実装（1分間に同一IPから10回まで）。`X-RateLimit-*` ヘッダーと `Retry-After` を返す。

### [x] 3. エラーメッセージの細分化

`navigator.onLine` 確認を最優先、401/413/429/529/5xx を個別ハンドリング。

### [x] 9. 重複送信防止

`sessionStorage` に実行中フラグを保持。チェックボタンクリック時とリロード後の両方で検証。

### [x] 10. unclear 項目の個別再チェック

`reCheckUnclearItems()` を実装。`displayResults()` 内で unclear 件数が 1 件以上の場合のみ「判定不可項目を再チェック」ボタンを表示し、unclear 項目だけを再送して結果を上書き保存。

## 優先度 中（利便性の向上）

### [x] 4. チェックモデルの選択肢追加

`MODELS` 定数で Opus 4.7 / Sonnet 4.5 / Haiku 4.5 を選択可能に。`localStorage` に選択モデルを保存。

### [x] 5. 結果の印刷・PDF 出力

`window.print()` + `@media print` 専用 CSS を実装。結果カードのみが印刷される。

### [x] 6. チェック結果の要約バッジ

`✓ 記載あり N` / `✗ 記載なし N` / `? 判定不可 N` のカラーバッジを結果上部に表示。

### [x] 11. 履歴の検索・フィルター

カテゴリ / 判定結果 / 日付範囲 / キーワード（カテゴリ・メモ）でフィルター可能。

### [x] 12. チェック結果へのメモ追加

履歴レコードに `memo` フィールドを持たせ、結果画面で編集可能。CSV エクスポートにも含まれる。

### [x] 13. API 利用コスト表示

レスポンスの `usage` から入出力トークン数を取得し、モデル別単価で概算コスト（USD / JPY）を表示。

### [x] SP-01. ドロップゾーン文言のデバイス切り替え

`navigator.maxTouchPoints` と `@media (hover: none)` で判定し、タッチデバイスでは「タップして画像を選択」に切替。

### [x] SP-02. 結果テーブルのカード型化（スマホ対応）

`@media (max-width: 640px)` で結果・履歴テーブルをカード型レイアウトに変換。`data-label` 属性でラベル表示。

## 優先度 低（将来対応）

### [ ] 7. Vercel Analytics の有効化

ダッシュボードでの設定操作のため、コード変更不要。

### [ ] 8. Git 連携による自動デプロイ

Vercel ダッシュボードでの設定操作のため、コード変更不要。

### [x] 14. チェック項目のカスタマイズ

`localStorage` ベースのカスタム項目管理 UI を実装。`getAllItems()` でルール項目に追加され、プロンプトに含まれる。区分は `custom` バッジで表示。

### [x] 15. 画像の向き補正（EXIF 対応）

外部ライブラリなしで EXIF Orientation タグを直接パースし、Canvas で回転補正。

### [x] 16. オフライン対応（Service Worker）

`sw.js` を追加し、静的アセット（HTML/CSS/JS）を cache-first でキャッシュ。API リクエストは常にネットワーク。
