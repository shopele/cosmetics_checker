# 改良案一覧

作成日: 2026-05-19

---

## 優先度 高（実用性・安定性に直結）

### 1. 画像圧縮の実装

現状はリサイズ（1920px）のみで、高解像度 JPEG は数 MB になる場合がある。Canvas で JPEG 品質を下げて送信データを削減し、API コスト削減・タイムアウト軽減につながる。

### 2. API エンドポイントのレート制限

現状の `api/check.js` には制限なし。短時間に連打すると Anthropic の API コストが急増する。`vercel-rate-limit` パッケージまたは簡易カウンターで 1IP あたり N 回/分 に制限する。

### 3. エラーメッセージの細分化

現状は API エラーを一律表示。429（レート超過）、529（過負荷）、ネットワーク切断を個別にハンドルしてユーザーに適切なメッセージを出す。

### 9. 重複送信防止

チェック実行中に連打できる状態になっている。ボタンを `disabled` にする処理は既存だが、ページリロード後の状態が残らない。実行中フラグを `sessionStorage` で保持して2重送信を完全防止する。

### 10. unclear 項目の個別再チェック

判定不可になった項目だけを対象に再度 API を呼び出せる機能。全項目を再送するより安価で、ピンポイントで精度を上げられる。

## 優先度 中（利便性の向上）

### 4. チェックモデルの選択肢追加

現状 `claude-opus-4-7` 固定。速度重視なら Haiku、精度重視なら Opus を選べるようにすると、利用シーンに合わせてコスト調整できる。

### 5. 結果の印刷・PDF 出力

CSV エクスポートはあるが、製品ごとにチェック結果を 1 枚で印刷できる機能は監査証跡として有用。`window.print()` + 専用 CSS で対応可能。

### 6. チェック結果の要約バッジ

「○項目 OK / △項目要確認 / ✕項目 NG」をカラーバッジで一目で示す（現状は表のみ）。

### 11. 履歴の検索・フィルター

現状は最新50件をそのまま表示。カテゴリ（化粧品/医薬部外品）・判定結果（OK/NG）・日付範囲でフィルターできると、過去の結果を参照しやすくなる。

### 12. チェック結果へのメモ追加

AI 判定後に担当者が手動でコメントを追記できるフィールド。「実物確認済み」「要修正」などの業務メモを CSV にも含めて出力する。

### 13. API 利用コスト表示

レスポンスの `usage` フィールド（input/output tokens）から概算コストを計算して結果画面に表示。コスト管理に役立つ。

### SP-01. ドロップゾーン文言のデバイス切り替え

「ドラッグ＆ドロップ」とあるがスマホではタップのみ有効（動作はする）。`navigator.maxTouchPoints` でタッチデバイスを判定し、文言をスマホ時は「タップして選択」に切り替える。CSS `@media (hover: none)` との併用でホバー効果もスマホでは非表示にできる。

### SP-02. 結果テーブルのカード型化（スマホ対応）

結果・履歴テーブルが 5〜6 列あるため、スマホでは横スクロールが必要になる場合がある。`@media (max-width: 640px)` でテーブルをカード型レイアウトに切り替えることで解消できる。

## 優先度 低（将来対応）

### 7. Vercel Analytics の有効化

現状 Not Enabled。ダッシュボードから 1 クリックで有効にでき、利用頻度・エラー傾向の把握に役立つ。

### 8. Git 連携による自動デプロイ

現状は手動 `vercel --prod`。GitHub リポジトリを Vercel に連携すると `main` ブランチへのプッシュで自動デプロイされる。

### 14. チェック項目のカスタマイズ

薬機法以外の社内基準（自社表示ルールなど）を追加できるカスタム項目欄。`rules.js` に追記する運用でもよいが、UI から設定できるとノンエンジニアでも対応できる。

### 15. 画像の向き補正（EXIF 対応）

スマホで撮影した画像は EXIF の向き情報があり、ブラウザによっては正しく表示されても Canvas では回転してしまう。`exifr` ライブラリで補正すると判定精度が上がる。

### 16. オフライン対応（Service Worker）

ネットワーク切断時にアプリ自体が開かなくなるのを防ぐ。静的ファイルをキャッシュしておけば、API は使えなくても過去の履歴は確認できる。
