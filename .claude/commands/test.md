test-agent として動作し、化粧品・医薬部外品 表記チェックアプリのテストを実施してください。

## 前提情報
- テスト仕様書: `docs/test/spec.md` を参照
- 実装コード: `index.html`, `css/style.css`, `js/rules.js`, `js/app.js`

## テスト実施内容

### 1. 静的コード検証
- `js/rules.js` の内容を読み、医薬部外品10項目・化粧品7項目が正しく定義されているか確認
- 根拠条文（薬機法の条番号）が正確に記載されているか確認
- `index.html` の構造を確認（必須要素の存在確認）
- `js/app.js` の主要関数の実装を確認

### 2. コードレビューによる機能確認
以下の各機能の実装コードをレビューし、正しく実装されているか確認する：
- [ ] 画像アップロード処理
- [ ] カテゴリ切り替え
- [ ] APIキーの保存・読み込み
- [ ] Claude API 呼び出し（ヘッダー・本文の正確性）
- [ ] プロンプト設計（チェック項目の網羅性）
- [ ] レスポンスのパース処理
- [ ] 結果表示ロジック
- [ ] 履歴保存・読み込み
- [ ] CSVエクスポート
- [ ] エラーハンドリング

### 3. バグの記録
発見したバグは以下の形式で記録する：
```
## バグ報告 #[連番]
重大度: Critical / High / Medium / Low
カテゴリ: 機能バグ / UI不具合
内容: [具体的な問題]
対象ファイル・行: [ファイルパス:行番号]
修正方針: [どう直すべきか]
```

## 完了条件
テスト結果を `docs/test/results.md` に記録し、Critical/High バグがあれば build-agent に修正を依頼すること。
