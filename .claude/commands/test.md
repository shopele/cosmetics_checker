test-agent として動作し、化粧品・医薬部外品 表記チェックアプリのテストを実施してください。

## 【必須ルール】

**このスキルはコード変更後に必ず実行すること。全テストが PASS になるまで build-agent による修正 → 再テストを繰り返し、FAIL が残ったままにしない。**

## 前提情報

- テスト仕様書: `docs/test/spec.md` を参照
- 実装コード: `index.html`, `css/style.css`, `js/rules.js`, `js/app.js`, `api/check.js`
- テスト結果記録先: `docs/test/results.md`（末尾に追記）

## テスト実施内容

### 1. 静的コード検証

- 変更されたファイルを読み込み、構文エラー・ロジックエラーがないか確認
- 新機能の実装が要件通りか確認
- XSS対策（`escapeHtml()` の適用）を確認
- null/undefined の安全なハンドリングを確認

### 2. 機能テスト（コードレビューによる確認）

変更に関連する以下の機能を確認：
- [ ] 新機能の正常系動作
- [ ] 新機能のエッジケース（空配列・null・undefined・空文字）
- [ ] 既存機能へのデグレードがないか
- [ ] エラーハンドリング

### 3. バグ報告フォーマット

```markdown
## バグ報告 #[連番]

**重大度**: Critical / High / Medium / Low
**カテゴリ**: 機能バグ / UI不具合 / セキュリティ / パフォーマンス
**再現手順**:
1.
2.
**期待動作**:
**実際の動作**:
**対象ファイル・行**: [ファイルパス:行番号]
**修正案**:
```

### 4. テスト結果の記録フォーマット

`docs/test/results.md` の末尾に以下を追記：

```markdown
---

## [機能名] テスト（コミット [hash]）

実施日: [日付]

| # | テスト項目 | 判定 | 備考 |
|---|---|---|---|
| 1 | ... | PASS/FAIL | |

### 総合判定: PASS / FAIL

[判定理由・備考]
```

## 完了条件

- [ ] テスト結果を `docs/test/results.md` に追記済み
- [ ] **全テスト PASS**（FAIL がある場合は build-agent に修正を依頼し再テスト）
- [ ] 結果をコミット・プッシュ済み（`git commit -m "test: ..."` / `git push origin main`）
